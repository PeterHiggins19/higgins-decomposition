- Scale range: 10⁻¹⁸ m (QCD quarks) to 10²⁶ m (Hubble radius) — 44 orders of magnitude

**Feed-Forward Governance Architecture Established:**

The Hˢ repository adopts a feed-forward controlled advancement model. The structure
is analogous to a PID control loop applied to knowledge management:

**Proportional (P) — Current State:**
The mainstream pipeline and active experiments. Responds to current findings.
New data enters here, gets diagnosed against current standards, and either
advances to mainstream or is routed to archive.

**Integral (I) — Archive / Long-Term Memory:**
All past experiments, superseded methods, and retired approaches are archived
as JSON snapshots. They are never deleted — they provide the long-term stability
that prevents oscillation. A full refresh of the archive is included in every
feed-forward cycle to check for patterns that became faint but remain relevant.
All documents eventually become JSON in archive, pruned from mainstream but
preserved for integral reference.

**Derivative (D) — Trajectory / Waypoints:**
The executive summary waypoint system provides the derivative term. Each
waypoint declares the current position, the next target, the gate conditions,
and the drift boundary. The anti-drift compass uses the declared trajectory
to pre-correct for deviation before it accumulates.

**Feed-Forward (FF) — Predictive Bias:**
New information is biased toward forward use. The system is forward-biased:
new data introduced is diagnosed for current use, not retrospective validation.
Historical data is checked but not privileged. The future state (predicted by
waypoints) influences acceptance criteria. Information that aligns with the
declared trajectory gets lower resistance; information that contradicts it gets
higher scrutiny — but is never rejected without investigation, because
contradictions are where discoveries live.

**Acceptance Criteria for Pipeline Inclusion:**
1. New data must pass all 4 input guards (D≥2, N≥5, no NaN/Inf, scale sanity)
2. New data must produce a valid extended pipeline run (no crashes)
3. Results must be documented with full diagnostic codes
4. Results must be compared against the reference standard library
5. Any new constants, carriers, or methods must survive regression against
   all existing experiments (HIVP chain principle)
6. Findings are classified by claim tier before publication

**Archive Protocol:**
- Superseded methods → archived as JSON with timestamp and reason
- Retired experiments → archived with final results and comparison to replacement
- Old terminology → archived in the NOT lists with retirement date
- Historical documents → JSON snapshot in archive, pruned from mainstream
- Archive refresh → included in every major waypoint review
